  1. General information:

    - Click on Backlog menu to go to Backlog page

    - This page is used for moving user story (US) between Product Backlog and Sprint

    - When an US is assinged to Sprint, this US will be called Issue

  2. Detail information:
    1. Filter Options:

      - Choose the project, team and sprint that you will work with it.

      - New Backlog Item: add new backlog item for Product Backlog.

    2. Search area

      1: Link to show undone tasks.

      2: when there are more then 15 user stories, roll down the mouse to see more.

      3. User can drag and drop to move US between Product backlog and Sprint Backlog.

      *Note:

      If the task is not done in one sprint, it will have gray color. And you can see the history of the task ( When it begin? which sprint is it implemented?).
